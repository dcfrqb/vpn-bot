# CRS VPN Telegram Bot

Telegram-бот для продажи и управления VPN-подписками. Позволяет пользователям покупать подписки, получать конфигурации VPN и управлять аккаунтом без необходимости регистрации на веб-сайте.

## Overview

Бот интегрирован с платежными системами (YooKassa, криптоплатежи) и VPN-сервисом Remna API. Обрабатывает платежи, создает подписки, синхронизирует данные с внешними сервисами и предоставляет административную панель для управления системой.

## Key Features

- **Покупка подписок** — выбор тарифного плана и оплата через YooKassa или USDT
- **Автоматическая выдача конфигураций** — получение VPN-конфигурации после успешной оплаты
- **Управление подписками** — просмотр статуса, срока действия, обновление данных
- **Административная панель** — статистика пользователей, платежей, управление системой
- **Синхронизация с Remna API** — автоматическая синхронизация пользователей и подписок
- **Кэширование** — оптимизация производительности через Redis
- **Webhook обработка** — асинхронная обработка платежных уведомлений

## Tech Stack

- **Python 3.11** — основной язык разработки
- **aiogram 3.x** — фреймворк для Telegram Bot API
- **FastAPI** — webhook сервер для обработки платежей
- **PostgreSQL** — основная база данных
- **Redis** — кэширование и FSM storage
- **SQLAlchemy 2.x** — ORM с асинхронной поддержкой
- **Alembic** — миграции базы данных
- **Docker & Docker Compose** — контейнеризация и оркестрация

## Quick Start

### Prerequisites

- Docker и Docker Compose
- Bash

### Setup

1. Клонировать репозиторий:
```bash
git clone <repository-url>
cd vpn-bot
```

2. Настроить конфигурацию:
```bash
cp config.example.env .env
# Отредактировать .env и заполнить обязательные параметры
```

3. Запустить:
```bash
docker compose up -d
```

Система автоматически:
- Создаст и запустит PostgreSQL и Redis
- Применит миграции базы данных
- Запустит Telegram бота и webhook сервер

### Configuration

Обязательные параметры в `.env`:
- `BOT_TOKEN` — токен Telegram бота
- `ADMINS` — список ID администраторов
- `YOOKASSA_SHOP_ID`, `YOOKASSA_API_KEY` — параметры YooKassa
- `REMNA_API_BASE`, `REMNA_API_KEY` — параметры Remna API

Полный список параметров см. `config.example.env`.

## Notes

- Бот работает в режиме polling по умолчанию. Для webhook режима укажите `TELEGRAM_WEBHOOK_URL`
- Данные хранятся не менее 5 лет для соответствия требованиям финансового учета
- Система спроектирована для горизонтального масштабирования
- Все секретные данные хранятся в `.env` (не коммитится в Git)
