# CRS VPN Telegram Bot

Telegram-бот для продажи и управления VPN-подписками. Автоматизирует процесс покупки подписок, обработки платежей и выдачи VPN-конфигураций через удобный интерфейс Telegram без необходимости регистрации на веб-сайте.

## Описание проекта

Система решает задачу автоматизации продажи VPN-подписок через Telegram, интегрируя платежные системы (YooKassa, криптоплатежи) и VPN-сервис Remna API. Бот обрабатывает полный цикл: от выбора тарифного плана до автоматической выдачи конфигурации после успешной оплаты.

Проект спроектирован как отдельный микросервис в рамках более крупной системы управления VPN-инфраструктурой. Архитектура обеспечивает горизонтальное масштабирование, изоляцию компонентов и возможность независимого развертывания.

## Цели и задачи

### Основная цель
Автоматизация продажи и управления VPN-подписками через Telegram с минимальным участием администратора.

### Практические задачи
- **Для пользователей**: покупка подписок, получение VPN-конфигураций, управление аккаунтом через Telegram
- **Для администраторов**: мониторинг пользователей и платежей, управление подписками, статистика системы
- **Для системы**: синхронизация данных с VPN-панелью, обработка платежей, автоматическое обновление статусов подписок

### Целевая аудитория
- **Пользователи**: клиенты, приобретающие VPN-подписки
- **Администраторы**: операторы системы, управляющие подписками и пользователями

## Функциональные возможности

### Возможности для пользователя
- Выбор тарифного плана VPN-подписки
- Оплата через YooKassa (банковские карты) или USDT (криптовалютные платежи)
- Автоматическое получение VPN-конфигурации после успешной оплаты
- Просмотр статуса активной подписки, срока действия
- Обновление данных подписки и синхронизация с VPN-панелью
- Просмотр истории платежей и подписок

### Возможности для администратора
- Просмотр статистики пользователей, платежей, подписок
- Управление подписками пользователей
- Обработка запросов на выдачу VPN-доступа
- Мониторинг синхронизации с Remna API
- Просмотр логов и диагностика системы

### Интеграции
- **Remna API**: синхронизация пользователей и подписок, получение VPN-конфигураций
- **YooKassa API**: обработка банковских платежей, webhook-уведомления
- **Telegram Bot API**: взаимодействие с пользователями через Telegram
- **FastAPI Webhook Server**: асинхронная обработка платежных уведомлений

### Роль репозитория vpn-bot

Данный репозиторий содержит сервис Telegram-бота, который является отдельным компонентом системы. Бот работает независимо от других сервисов и взаимодействует с внешними API через четко определенные интерфейсы.

### Взаимодействие компонентов

1. **Telegram Bot Service**: обрабатывает команды и callback-запросы от пользователей, управляет UI через систему экранов и навигации
2. **FastAPI Webhook Server**: принимает уведомления от YooKassa о статусе платежей, обрабатывает их асинхронно
3. **PostgreSQL**: хранит данные пользователей, подписок, платежей, обеспечивает целостность данных через транзакции
4. **Redis**: кэширует часто запрашиваемые данные, хранит состояние FSM для диалогов с пользователями
5. **Remna API**: источник истины для VPN-конфигураций, синхронизация пользователей и подписок
6. **YooKassa API**: обработка платежей, получение статусов транзакций

### Асинхронная модель

Все компоненты построены на асинхронной архитектуре:
- Использование `asyncio` и `aiogram` для асинхронной обработки запросов
- Асинхронные HTTP-клиенты для взаимодействия с внешними API
- Асинхронный доступ к базе данных через SQLAlchemy 2.x с `asyncpg`
- Неблокирующая обработка webhook-уведомлений

### Масштабируемость

Архитектура поддерживает горизонтальное масштабирование:
- Stateless-дизайн бота (состояние хранится в Redis)
- Разделение на независимые сервисы (bot, webhook-api)
- Возможность запуска нескольких инстансов бота с общим Redis
- Изоляция компонентов через Docker-контейнеры

## Запуск и развертывание

### Требования

- **Docker** 20.10+ и **Docker Compose** 2.0+
- **Bash** для выполнения скриптов

Все зависимости устанавливаются автоматически через Docker.

### Переменные окружения

Создайте файл `.env` на основе `config.example.env`:

```bash
cp config.example.env .env
```

Обязательные параметры:

- `BOT_TOKEN` — токен Telegram бота от @BotFather
- `ADMINS` — список ID администраторов (через запятую)
- `YOOKASSA_SHOP_ID`, `YOOKASSA_API_KEY` — параметры YooKassa
- `YOOKASSA_WEBHOOK_SECRET` — секрет для проверки webhook-уведомлений
- `REMNA_API_BASE`, `REMNA_API_KEY` — URL и ключ Remna API
- `DATABASE_URL` — URL подключения к PostgreSQL (опционально, по умолчанию используется настройка из docker-compose.yml)
- `REDIS_URL` — URL подключения к Redis (опционально, по умолчанию используется настройка из docker-compose.yml)

Опциональные параметры:

- `TELEGRAM_WEBHOOK_URL` — URL для webhook-режима Telegram (если не указан, используется polling)
- `YOOKASSA_WEBHOOK_URL` — URL для webhook-уведомлений YooKassa
- `WEBHOOK_API_PORT` — порт для FastAPI webhook сервера (по умолчанию 8001)
- `CRYPTO_USDT_TRC20_ADDRESS` — адрес кошелька для криптоплатежей

### Локальный запуск

#### Быстрый старт

```bash
# Выполнить все операции: сборка → тесты → запуск
./run_all.sh all
```

#### Поэтапный запуск

```bash
# 1. Сборка Docker образов
./run_all.sh build

# 2. Запуск приложения
./run_all.sh run
```

#### Через Docker Compose напрямую

```bash
docker compose up -d
```

При запуске автоматически:
- Создаются и запускаются контейнеры PostgreSQL и Redis
- Применяются миграции базы данных
- Запускается Telegram бот
- Запускается FastAPI webhook сервер

```bash
# Статус контейнеров
```


## Безопасность

### Работа с токенами и ключами

- Все секретные данные хранятся в `.env` файле, который не коммитится в Git
- Токены и ключи передаются через переменные окружения
- Использование `pydantic-settings` для валидации конфигурации

### Webhook-безопасность

- Проверка подписи webhook-уведомлений от YooKassa через `YOOKASSA_WEBHOOK_SECRET`
- Валидация данных платежей перед обработкой
- Логирование всех webhook-запросов для аудита

### Ограничения доступа

- Административные команды доступны только пользователям из списка `ADMINS`
- Проверка прав доступа через `AuthMiddleware`
- Изоляция данных пользователей на уровне базы данных

### Общие меры безопасности

- Использование параметризованных запросов для предотвращения SQL-инъекций
- Валидация всех входящих данных через Pydantic
- Логирование операций для отслеживания подозрительной активности
- Хранение данных не менее 5 лет для соответствия требованиям финансового учета
- Использование HTTPS для всех внешних API-вызовов

## Масштабирование и развитие

### Архитектурные решения для масштабирования

- **Stateless-дизайн**: состояние хранится в Redis, что позволяет запускать несколько инстансов бота
- **Асинхронная архитектура**: неблокирующая обработка запросов обеспечивает высокую производительность
- **Разделение сервисов**: bot и webhook-api работают независимо, могут масштабироваться отдельно
- **Кэширование**: Redis используется для кэширования часто запрашиваемых данных
- **Миграции БД**: Alembic обеспечивает версионирование схемы базы данных

### Возможные направления развития

- **Горизонтальное масштабирование**: запуск нескольких инстансов бота за балансировщиком нагрузки
- **Мониторинг и метрики**: интеграция с системами мониторинга (Prometheus, Grafana)
- **Расширение платежных систем**: добавление поддержки других платежных провайдеров
- **Мультиязычность**: поддержка нескольких языков интерфейса
- **Уведомления**: расширение системы уведомлений (email, push)
- **Аналитика**: детальная аналитика пользователей и платежей
- **API для внешних сервисов**: предоставление REST API для интеграции с другими системами
